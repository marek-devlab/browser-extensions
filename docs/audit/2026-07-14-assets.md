# Аудит `extensions/assets` (Asset Inspector) — 2026-07-14

Read-only аудит второй волны. Предмет: `extensions/assets/**` (реализация из моков в реальную логику), манифесты Chrome MV3 / Firefox MV2 в `.output/`.
Контекст прочитан: `docs/design/assets.md`, `TODO.md` §J (включая 🔴-инварианты), `PLAN-2.md` §4, `docs/audit/2026-07-14-blur.md` (планка формата).
Метод: чтение всего кода + grep на запрещённые сети/сингулярности + проверка обоих собранных манифестов. CONFIRMED / PLAUSIBLE проставлены.

## (a) Вердикт

**Блокеров нет. Экзистенциальные инварианты «инспектор, не загрузчик» держатся В КОДЕ, а не в комментариях.** Это главное, что нужно было доказать, и оно доказано:

1. **Ноль сети по показываемым URL.** Grep по `extensions/assets/**` (без `.output`): **ни одного** `fetch` / `XMLHttpRequest` / `new Image` / `.src =` / `createObjectURL` / `toBlob` / `toDataURL` / `sendBeacon` / `WebSocket` / `EventSource`. Единственные совпадения — комментарии, которые это ЗАПРЕЩАЮТ. Превью — `canvas.drawImage(theElement)` по уже декодированному браузером элементу (`inspector.content.ts:556`), и `toDataURL`/`toBlob` не вызываются нигде → пути к байтам физически нет, tainted-canvas даже не бросит.
2. **Никакой файл не пишется.** Экспорт карточки — только `navigator.clipboard.writeText` (`copyAsJsonButton` `:1140`). Нет `downloads`, нет `<a download>`, нет `createObjectURL`.
3. **Нет парсинга `.m3u8`/`.mpd`.** `mediaFeedRequests` (`resource-timing.ts:141`) только ФИЛЬТРУЕТ записи Resource Timing по `initiatorType`; манифест может ПОЯВИТЬСЯ в списке как факт о странице, но не открывается и не парсится. Карточка это прямо проговаривает (`:1006`).
4. **Нет `webRequest`, `<all_urls>`, `debugger`.** Оба манифеста (Chrome MV3 и Firefox MV2): `permissions` = ровно `["activeTab","scripting","storage","contextMenus"]`, `content_scripts:[]`, `host_permissions` **удалён** build-хуком (`wxt.config.ts:51`) — проверил в собранных файлах, паттерна `*://*/*` там нет. `web_accessible_resources` нет → страница не может фингерпринтить установку. `data_collection_permissions:["none"]`.
5. **Нет waterfall / оси времени / суммы байтов** (граница с `perf`). Перевес считается в ПИКСЕЛЯХ (`inspect.ts:445`), панель явно исключает waterfall/timing/Vitals (`panel/App.tsx:20`). «Перевес» — единственный выносимый вердикт.
6. **В листинге/UI нет слов download/save/grabber/ripper** — grep даёт только `cursor: grab` (CSS) и запрещающие комментарии.

Честность чисел соблюдена: `transferSize` для cross-origin без TAO → `null` → «не измерен», НИКОГДА не `0` (`resource-timing.ts:21`, `weightOf` `:28`, проносится до DOM в `propsSection` `:878`). Статус 0→`null`→«не измерен» (`:96`). Семантика переполнения Resource-Timing-буфера правильная — браузер роняет **новые** записи и держит ранние; текст предупреждения это говорит верно (`resource-timing.ts:146`, `bufferCallout` `:684`). `javascript:`-URL из `srcset` умирает на `isOpenable` (протокол проверяется ДО присвоения `href`, `inspector.content.ts:640`, `inspect.ts:398`). Карточка — closed shadow root, ноль `innerHTML` (всё `h()`→`textContent`). Мобильный паритет реальный: попап-кнопка «Point to an element» инжектит тот же тач-picker (`popup/App.tsx:101` → `background.ts:73`), контекстное меню и хоткей ведут туда же.

Находки — заметки, не дефекты категории.

## (d) 💡 УЛУЧШЕНИЯ / заметки

- **Попап никогда не сообщает о РЕАЛЬНОМ переполнении буфера.** `readCounters` (`popup/App.tsx:51`) — это отдельный `executeScript({func})`, и он хардкодит `overflowed:false` (`:66`); флаг реального переполнения (`resourcetimingbufferfull`) живёт на стейте оверлея (`inspector.content.ts:266`), а не в глобале, который может прочитать функция попапа. Итог: попап предупреждает только по эвристике «≥85% лимита» (`:108`), а поле `overflowed` в этом пути мёртво. Честно по духу (дизайн §5.11 и оперирует 85%), но `PageCounters.overflowed` вводит в заблуждение — либо публиковать флаг переполнения на изолированном глобале (как `__assetsInspectorBufferLimit`), либо убрать поле. PLAUSIBLE (низкая важность).
- **`background.ts:24` инжектит только через `browser.scripting.executeScript`, без фолбэка на `tabs.executeScript`.** В export аналогичный `inject.ts` фолбэк есть. Firefox MV2 поддерживает `scripting` с FF 109, так что риск мал, но если сборка попадёт на старый Firefox — picker-путь тихо ничего не сделает (catch проглатывает). Симметрии ради стоит добавить фолбэк.
- **`elementLabel` (`element-picker.ts:88`) собирает строку `<${tag} class="${cls}">`** и кладёт её через `textContent` (не `innerHTML`), так что это не XSS — но выглядит как HTML и может смутить при копировании в JSON. Косметика.
- **`inspect.ts:328` `closedShadow`-эвристика** (custom-element без light-DOM детей и с высотой) — честно помечена как условие, не вывод; ложные срабатывания возможны (пустой web-component без shadow). Дизайн это допускает (§4.7). Не трогать, просто отметить.

## (e) ✅ Проверено и в порядке (не переделывать)

- **Все 🔴-инварианты §J держатся в коде** (см. вердикт, пп. 1–6). Это проверено grep'ом и чтением, не по комментариям.
- **Манифесты (Chrome MV3 + Firefox MV2).** `permissions` минимальны, `content_scripts:[]`, `host_permissions` вырезан хуком (в собранных файлах паттерна нет), `web_accessible_resources` нет, `commands` (Alt+Shift+A) не даёт варнинга, `devtools_page` — ключ, не разрешение. `gecko.id=assets@blockaly.com`, `gecko_android:{}`, `data_collection_permissions:["none"]`. Ноль install-предупреждений.
- **`registration:'runtime'` для `inspector.content.ts`** (`:1256`) — скрипт НЕ в `content_scripts` манифеста, инжектится по жесту. Файл существует в `/content-scripts/inspector.js` для инъекции, но не web-accessible → страница не может его загрузить и по факту установки фингерпринтить.
- **Честность веса/статуса/редиректов.** `resolveTransferSize` различает три случая: `>0` реальный, `0` при `decodedBodySize>0` — кэш-хит (0 байт действительно прошло), `null` — cross-origin без TAO (неизмеримо) (`resource-timing.ts:21`). `redirectStateOf` даёт три РАЗНЫХ факта (`occurred`/`none`/`unknown`), «unknown» никогда не рендерится как «нет редиректов» (`:86`). MIME — `guessed-extension` явно маркируется «by file extension» (`:182`, `identityLine` `:539`); `data:` — точный.
- **Переполнение RT-буфера.** Семантика верна (роняет НОВЫЕ, держит ранние), `raiseBuffer` вызывается ПЕРВЫМ делом в `main()` до UI (`:1264`), и текст предупреждения честно говорит, что поднятие лимита не воскрешает уже отброшенное.
- **Picker (копия из adblock + расширения).** Единый `AbortController` → один `abort()` = полный teardown, курсор и оверлей восстанавливаются (`inspector.content.ts:322`). Capture-фазное глушение `mousedown/mouseup/click/auxclick/dblclick/submit` — пик не уходит по ссылке (`element-picker.ts:340`). Пирсинг OPEN shadow через `composedPath()[0]` (`:297`), closed root недостижим и честно назван. Клавиатура (↑↓←→ / `[` `]` стек / `R` ближайший ресурс / Enter / Esc), тач (tap+Confirm ≥44px). SPA-churn: ring едет за элементом по rAF, нулевой rect → drop (`:207`). Double-injection идемпотентна через `ACTIVE_FLAG`+`overlay` (`:191`), listener-лик закрыт отдельным `LISTENER_FLAG` (`:1271`). `MutationObserver` на удаление узла помечает карточку снапшотом, не закрывает (`:403`).
- **XSS/инъекции.** Ноль `innerHTML`/`outerHTML`/`insertAdjacentHTML`/`dangerouslySet`/`eval`. `h()` кладёт page-данные (alt, URL, srcset, MIME, cell text) только в `textContent`/`setAttribute` (`:68`). Стили — статический `CSSStyleSheet.replaceSync`, без интерполяции URL (`:211`). `openLink` присваивает `href` только после `isOpenable` (http/https). Панель — React (авто-эскейп), picked-значение из `inspectedWindow.eval` через page-глобал — недоверенные данные, рендерятся текстом, href из них не строится (`panel/App.tsx:220`). `srcset`-probe (`srcset.ts:178`) ставит `.style.width = length` (сеттер одного свойства не пускает вторую декларацию → нет CSS-инъекции), живёт в closed shadow, `contain:strict`, удаляется сразу, сети ноль.
- **Closed shadow root** на `documentElement` (`:210`) — страница не читает `host.shadowRoot` и не подменяет карточку.
- **Мобильный паритет.** Попап-кнопка → `assets:openPicker` → `injectInspector` (`background.ts:73`); хоткей и контекстное меню туда же. Picker работает тачем. DevTools-панель — строгий апгрейд, ничего в карточке от неё не зависит (`panel/App.tsx:34`).
- **DevTools-панель.** Не может использовать `scripting`/`activeTab` (грабли `seo`) — использует `inspectedWindow.eval` с фиксированными source-строками (`PICKER/POLL/STOP_SOURCE`), poll ограничен и всегда чистится (`:100`), `onNavigated` роняет HAR и рисует «страница устарела» (`:72`). Показывает только то, чего в попапе быть не может (инициатор/редиректы/точный MIME/статус); waterfall/байты/Vitals отсутствуют. Bulk-копирование запрещено, только построчно (`:197`) — граница «не харвестер».
- **Background без состояния.** Только меню + инъекция по жесту; `srcUrl` — единственное, что уходит в страницу, и только по явному действию (`background.ts:22`). `tryContextMenuMatch` матчит `srcUrl` по нормализованному URL, один матч → карточка, иначе picker (`inspector.content.ts:308`).
- **Сеть.** Ноль. Нет `fetch`/beacon/аналитики/update-check во всём расширении.

## (f) Что НЕ покрыто (проверял чтением)

1. **`overflowed` в попапе** (§d) — нет пути протестировать реальное переполнение из попапа, т.к. флаг там мёртв.
2. **Firefox-фолбэк инъекции** (§d) — `background.ts` без `tabs.executeScript`, на старом FF не сработает; теста нет.
3. **MSE/DRM-ветка** (`inspect.ts:160`) — `blob:`/пустой `currentSrc` при `videoWidth>0`; `video.mediaKeys!=null`→DRM. Живьём тестируется только на реальном плеере; эвристика запросов помечена «heuristic» в UI.
4. **`srcset`-реконструкция vs браузер** — `modelDisagrees` выносится громко (`srcset.ts:94`), но расхождение (кэш/Data-Saver) тестами не воспроизводится.
5. **Closed-shadow эвристика** (`inspect.ts:328`) — ложные срабатывания не покрыты.
