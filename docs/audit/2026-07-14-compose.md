# Аудит `extensions/compose` (Markdown Workbench) — 2026-07-14

Первый аудит. Расширение только что переведено с моков на реальную логику. Read-only: код не менялся.
Предмет: `extensions/compose/**` (utils/{sanitize,markdown,clipboard,convert,regex.worker,regex-client,storage,surface,…}, components/**, entrypoints/**), манифесты Chrome MV3 / Firefox MV2.
Контекст прочитан: `docs/design/compose.md`, `TODO.md` §L, `PLAN.md` (Часть II) §6/§10.6, формат дома — `docs/audit/2026-07-14-blur.md`.
Атака на санитайзер (приоритет №1) велась целенаправленно: mXSS foreign-content, DOM clobbering, `srcdoc`/`base`, конфиг-дыры, путь сериализатора. Версия DOMPurify — **3.4.12** (актуальная, без известных байпасов).

---

## (a) Вердикт

**К сабмиту готово; граница безопасности держится под враждебным конфигом.** Это сильная, дисциплинированная реализация. Санитайзер построен **правильно по конструкции**, а не «на комментариях»: единственная точка `string→DOM` во всём коде (`sanitizeToFragment`), `RETURN_DOM_FRAGMENT` → `replaceChildren` (узлы, не строка), закрытый Shadow DOM, allow-list (не block-list), `svg`/`math`/`template`/`style`/`iframe`/`base` в `FORBID_TAGS` (закрывает mXSS через foreign-content и re-parse `<style>`), `name`/`id` не в allow-list + `name` в `FORBID_ATTR` (закрывает DOM-clobbering), `ALLOWED_URI_REGEXP` только `https?|mailto` (режет `javascript:`/`data:`/`blob:`/`srcdoc`), `style`-атрибут запрещён (закрывает clickjacking маской). Путь сериализатора (`serializeFragment`) идёт **только** DOM→строка→буфер обмена и **нигде не репарсится** в расширении — sink'а нет. Regex — **только** в терминабельном воркере с реальным `terminate()` и респауном; главный поток `new RegExp(userInput)` не трогает даже для валидации. Черновики — `storage.local` с Web-Lock RMW и append-merge (гонка «фон дописал / панель перетёрла» решена по-настоящему); «saved» показывается только после резолва `setValue`; QuotaExceeded — видимая ошибка, не тихая потеря. Сеть — ноль (`connect-src 'none'`, 0 `fetch`), эмодзи-данные лениво (`await import`), транслит — свои таблицы. Мобильный fallback (Android без sidebar) сделан по фиче-детекту, не по UA.

Блокеров нет. Одно ⚠️ реально стоит закрыть до того, как опираться на маркетинг «100% local, no network»: **превью автоматически грузит удалённые картинки** из `![](http://…)`, что превращает вставленный чужой markdown в трекинг-пиксель и делает сетевой запрос из привилегированной страницы — а `connect-src 'none'` даёт ложное чувство защиты (он не покрывает `img`).

## (c) ⚠️ ВАЖНОЕ

### V1. Превью автоматически грузит удалённые изображения → трекинг-пиксель и сеть из привилегированной страницы; «zero network» нарушено
`utils/sanitize.ts:28` (`img` в `ALLOWED_TAGS`), `:33` (`src` в `ALLOWED_ATTR`), `:38` (`ALLOWED_URI_REGEXP = /^(?:https?|mailto):/i` — `http(s)` разрешён), `:113-116` (`img` получает `loading=lazy`, `referrerpolicy=no-referrer`, но URL остаётся удалённым). Рендер: `markdown.ts:77` → `PreviewPane.tsx:80` `replaceChildren`.

`![x](http://tracker.example/p.png?id=abc)` в черновике markdown-it превращает в `<img src="http://tracker.example/p.png?id=abc">`, санитайзер это **пропускает** (схема `http` разрешена), и превью в закрытом Shadow DOM на **странице расширения** немедленно инициирует сетевой запрос к произвольному хосту при первом же рендере (debounce 120 мс, авто). Последствия:

1. **Нарушение заявленной приватности.** Манифест (обе сборки) в `description`: «100% local, no network». Дизайн — «Ноль сети». Автозагрузка удалённой картинки — это исходящий сетевой запрос, утечка IP/времени/факта просмотра черновика произвольному третьему лицу.
2. **CSP даёт ложную гарантию.** `content_security_policy … connect-src 'none'` (chrome-mv3 manifest) **не** покрывает `img-src`/`media-src` — директива `img-src` отсутствует, поэтому картинки грузятся. Ревьюер и разработчик, читающие «connect-src 'none'», решат, что сети нет.
3. **Вектор через вставку.** Черновик приходит и извне — контекстное меню «Добавить выделенное» (`background.ts:73`), «Вставить из буфера», шаблоны. Вставил чужой баг-репорт с картинкой-пикселем → пинганул трекер, ничего не нажимая.

**CONFIRMED по коду** (allow-list явно пропускает `img`+`http src`; загрузка удалённого `<img>` — штатное поведение браузера). Не «дефейс», но прямое противоречие центральному обещанию продукта.

**Фикс:** добавить в `extension_pages` CSP `img-src 'self' data:;` (и `media-src 'none'`) — это блокирует удалённую загрузку, сохраняя data-URI и локальные ресурсы, и делает обещание правдой на уровне платформы. Либо в `afterSanitizeAttributes` для `img` заменять удалённый `src` на плейсхолдер/`data:` и показывать «изображение не загружено (ноль сети)». Первое предпочтительнее — граница на CSP, а не на санитайзере.

> ✅ **ИСПРАВЛЕНО 2026-07-14.** В CSP обеих сборок добавлен `img-src 'self' data:`. Проверено на собранных манифестах: `script-src 'self'; object-src 'self'; img-src 'self' data:; connect-src 'none'`. Удалённый `<img>` теперь блокируется браузером — «ноль сети» правда на уровне платформы, локальные и data-URI картинки работают.

### V2. `serializeFragment` доверяет **именам** атрибутов без экранирования (defence-in-depth)
`utils/sanitize.ts:191-192`:
```ts
for (const a of Array.from(el.attributes)) {
  attrs += ` ${a.name}="${escapeAttr(a.value)}"`;   // a.name не валидируется
}
```
Значения экранируются (`escapeAttr`), теги — по allow-list (`:189`), но **имя** атрибута вставляется как есть. Сегодня это безопасно: узлы приходят из `DOMPurify.sanitize`, который отдаёт только атрибуты из `ALLOWED_ATTR` (безопасные идентификаторы). То есть эксплойта нет — но единственная защита имени атрибута находится в чужой библиотеке, а не в этом сериализаторе, который сам себя позиционирует как «escaping is ours» (`:176`). Строка уходит в буфер как `text/html` (`convert.ts:491`, `clipboard.ts:33-38`) и в расширении не репарсится — sink'а нет, поэтому это ⚠️-defence-in-depth, не дыра.

**Фикс:** пропускать только `a.name`, матчащий `/^[a-z-]+$/`, либо брать имя из своего же `ALLOWED_ATTR`. Дёшево и убирает единственную неявную зависимость сериализатора от внутренностей DOMPurify.

### V3. Конвертер GitLab — тождество без обещанного предупреждения о расхождениях GLFM
`utils/convert.ts:484-486`: для `github`/`gitlab` возвращается `body` дословно с `degradations: []`. Дизайн §6.2 обещает: «**GitLab: Идентичность + предупреждение о расхождениях GLFM** (§11)». Предупреждения нет — для GitLab CounterStrip покажет «совместимо» там, где GLFM (свой рендер `<details>`, свои расширения) может разойтись с превью «близко к GitHub». Это не потеря данных (текст цел), но мелкая нечестность UI против собственного дизайна.

**Фикс:** для `gitlab` добавить одну информ-деградацию «GLFM рендерит часть конструкций иначе, чем это превью» (не блокирующую копирование).

## (d) 💡 УЛУЧШЕНИЯ

- **`pushSnapshot`: ⚑ pre-destructive снимки никогда не вытесняются** (`storage.ts:102-119`). Если у одного черновика накопится > `historyLimit` именно ⚑-снимков (много «Заменить всё» подряд), буфер по этому черновику растёт неограниченно — медленный дрейф к 10 МБ `local`. Ввести отдельный потолок на ⚑ (напр. не больше `historyLimit`, вытесняя самые старые ⚑ после автосейвов).
- **`storageUsage` оценивает по `drafts`+`history`** при отсутствии `getBytesInUse` (`storage.ts:158-160`), игнорируя `templates`/`recentEmoji` — оценка чуть занижена. Мелочь, но «Занято: X из ~10 МБ» лучше считать по всем items.
- **`connect-src 'none'` без `img-src`** — см. V1; заодно добавить `style-src 'self' 'unsafe-inline'` осознанно (превью-CSS ставится `textContent` в `<style>` внутри shadow — работает, но фиксация директивы избавит от сюрпризов).
- **Транслит/счётчик**: `Intl.Segmenter` — фолбэк на `[...value]` (по code points) в `devdata`; здесь стоит проверить, что и `compose` считает графемы через `Intl.Segmenter` (дизайн §2.7 требует), а не `str.length`.
- **Regex `MATCH_CAP = 5000`** обрабатывается честно: `truncated` показывается («первые 5000 — уточните шаблон», `ToolDrawer.tsx:235`) и «Заменить всё (N)» несёт точное N — тихой частичной замены нет. ОК, ничего не менять.

## (e) ✅ Проверено и в порядке (не переделывать)

- **Санитайзер держится под атакой.** `dompurify@3.4.12`. `sanitizeToFragment` — единственная точка `string→DOM` (`sanitize.ts:133-158`); возвращает `DocumentFragment`, крепится `replaceChildren` (`PreviewPane.tsx:80`). `FORBID_TAGS` = `script,style,iframe,object,embed,form,link,meta,base,svg,math,template,noscript,button,select,textarea` (`:41-44`) — foreign-content mXSS (`svg`/`math`), re-parse `<style>`, `<base>`-hijack, `srcdoc` закрыты. DOM-clobbering закрыт: `id`/`name` не в `ALLOWED_ATTR`, `name` в `FORBID_ATTR` (`:32-35`, `:51`), а код превью не делает `getElementById`/чтения clobber-свойств. `<input>` форсится в `disabled checkbox` без `name` (`:118-125`). `<a>` получает `target=_blank` + `rel=noopener noreferrer nofollow` (`:107-111`). `style`-атрибут запрещён → маска-clickjack невозможна. Схемы URL — только `https?|mailto` (`:38`). Чекбоксы task-list эмитятся как `html_inline` и проходят **тот же** санитайзер (`markdown.ts:56-58`) — парсер не в доверии. `md`-конфиг: `linkify:false` (нет авто-href из текста), `typographer:false`, `html:true` только ради `<details>` (`markdown.ts:20-25`).
- **Путь сериализатора не реэнтерабелен.** `renderHtmlString` → `serializeFragment` (DOM уже санитизирован) → строка `text/html` в буфер (`markdown.ts:86-89`, `convert.ts:491`, `clipboard.ts:33-38`). Grep подтверждает: строка **нигде** не возвращается в DOM (нет `innerHTML`/`insertAdjacentHTML`/`setHTML`/`dangerouslySetInnerHTML` во всём `extensions/compose`). Реэнтерабельного mXSS-sink нет.
- **Превью в закрытом Shadow DOM.** `attachShadow({mode:'closed'})`, стиль через `textContent`, а не `innerHTML` (`PreviewPane.tsx:96-101`). Классы превью не достают до стилей расширения. Исключение парсера ловится и не роняет редактор (`:84-88`).
- **Regex — только в воркере, реально терминабельный.** `new RegExp(userInput)` живёт **исключительно** в `regex.worker.ts:87`; главный поток не компилирует пользовательский паттерн даже для валидации (`regex-client.ts:19-20`). Таймаут `regexTimeoutMs` → `worker.terminate()` + респаун (`regex-client.ts:101-109`, `:72-79`); `runRegex` резолвит, не реджектит; id-gate роняет устаревшие ответы; `MATCH_CAP` 5000 против взрыва памяти; `expand()` разворачивает `$…` вручную, **не** перезапуская regex (`regex.worker.ts:63-71`). ReDoS главный поток не вешает.
- **Черновики: гонки и потеря данных решены.** Всё в `local:` (`storage.ts:25-65`), никакого `sync`. RMW под Web-Lock через **все** контексты (панель + фон), с append-merge против клоббера фоновой дописи из контекстного меню (`useDraft.ts:98-157`, `storage.ts:79-83`). «Saved» — только после резолва `setValue` (`useDraft.ts:137-142`); QuotaExceeded — видимая ошибка, текст остаётся в редакторе (`:145-154`). `session:unsaved` зеркалит на `visibilitychange`/`pagehide` (закрытие sidebar убивает документ без beforeunload) и **предлагает** восстановление, а не автоприменяет (`:80-84`, `:207-233`). ⚡ Деструктивные операции берут ⚑-снимок до записи (`:333-359`). `version`+`migrations` объявлены.
- **Сеть/эмодзи/бандл.** 0 `fetch`/`XHR`/`WebSocket`; `connect-src 'none'` в обеих сборках. Эмодзи-данные — лениво `import('emojibase-data/...')` в `emoji.ts:51-52`, не в основном бандле; `Workbench.tsx` тянет только функцию `loadEmoji`. Транслит — свои таблицы (не npm).
- **Разрешения и single purpose.** `storage, contextMenus, clipboardWrite, activeTab (+ sidePanel в Chrome)` — 0 install-предупреждений. **Нет `scripting`**: контекстное меню читает `info.selectionText` напрямую, инъекции нет (`background.ts:20-24`, `:73`) — `activeTab` строго read-only. Одна `action`, один пункт меню (+ подпункт «как цитату»), нет `commands`, открывающих отдельный инструмент; regex/транслит/счётчик — дочерние компоненты редактора без своих точек входа. Инвариант §1.1 соблюдён. `clipboardWrite` используется ровно в `clipboard.ts`. Конверсия площадок — только на копировании, исходный markdown не переписывается (`convert.ts:6-9`, §4.5); «не терять молча» соблюдено (details→развёрнуто, table→код-блок, всё в `degradations`, `convert.ts:443-474`).
- **Мобильный (Firefox Android) сделан правильно.** `openEditor` фиче-детектит `sidePanel`→`sidebarAction`→**полная вкладка `workbench.html`** (`surface.ts:52-72`), без UA-снифинга; на Android, где sidebar нет, клик по иконке открывает вкладку, а не «ничего». S1 и S2 — один компонент через `@container` (дизайн §1.2).
- **Клавиатура/буфер.** Лестница копирования `ClipboardItem`(rich) → `writeText` → `execCommand` → честная панель ручного копирования (`clipboard.ts:25-79`); тихого «ничего не произошло» нет.

## (f) Что стоит покрыть тестами

1. **V1** — тест «превью не инициирует сетевой запрос на `![](http://…)`» (после фикса CSP — что `img-src` его блокирует). Сейчас утечка невидима.
2. **Санитайзер — батарея XSS/mXSS**: `<svg><foreignObject>`, `<math><annotation-xml>`, `<template>`, `<a href=javascript:>`, `<img src=x onerror=…>`, DOM-clobbering `<a id=… name=…>`, `srcdoc`/`<base>`, вложенности, к которым чувствителен DOMPurify. Ни одного нет — а это единственная граница компрометации расширения.
3. **Путь сериализатора** — тест, что `renderHtmlString` не создаёт узлы, отсутствующие в allow-list, и что `text/html` не реэнтрится в DOM.
4. **RMW-гонка черновиков** — параллельная запись из фонового контекстного меню и из панели: append не теряется (регресс §8.4).
5. **Regex-таймаут** — `(a+)+$` на длинном тексте: приходит `timeout`, редактор жив, следующий запрос идёт в свежий воркер.
6. **QuotaExceeded** — мок отказа `setValue`: статус «не сохранено», текст цел, `session:unsaved` содержит последнюю версию.
