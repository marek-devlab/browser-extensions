Манифесты (оба таргета) — без install-предупреждений: `host_permissions` нет, `content_scripts` нет, `web_accessible_resources` нет (значит, страница не может фингерпринтить установку через `chrome-extension://`-URL). `data_collection_permissions: ["none"]` для AMO верен. ## (c) ⚠️ ВАЖНОЕ

### V1. CSV-guard: лидеры TAB и CR — мёртвые записи (`trimStart` съедает их до проверки). CONFIRMED
`utils/csv-guard.ts:16` `DANGEROUS_LEADERS = new Set(['=','+','-','@','\t','\r'])`, `:37` `const t = raw.trimStart();`, `:39` `DANGEROUS_LEADERS.has(t[0]!)`.

`trimStart()` удаляет ведущие пробельные символы, **включая `\t` и `\r`**, ещё до того, как проверяется первый символ. Поэтому записи `'\t'` и `'\r'` в наборе никогда не могут сматчиться — они недостижимы. Прогон эскейпера это подтверждает:

```
"\t=1+1"  → risk=true  out="'\t=1+1"   (пойман через '=', НЕ через таб)
"\tSUM"   → risk=false out="\tSUM"      (таб срезан, 'S' не опасен → НЕ экранирован)
"\t"      → risk=false out="\t"         (пустой после trim → НЕ экранирован)
"\r=cmd"  → risk=true  out="'\r=cmd"    (пойман через '=')
```

**Насколько это опасно на деле — низко.** Реальный вектор исполнения (формула ПОСЛЕ ведущего таба/CR, `\t=1+1`) всё равно ловится по символу `=/+/-/@`, который `trimStart` обнажает. Не ловится только «таб/CR + не-формула» (`\tSUM`, одиночный `\t`) — а это не инъекция. То есть эксплойта нет.

Но: и комментарий в шапке файла (`:4` «A cell beginning with `= + - @` (or TAB/CR)»), и TODO §G, и `design.md` §8.3 прямо декларируют TAB/CR как самостоятельные опасные лидеры. Код это заявляет и структурно не делает — классическое «мёртвое условие, дающее ложную уверенность на ревью и в следующей правке».
**Фикс:** либо проверять первый символ ДО `trimStart` (или триммить только пробел/` `, не `\t`/`\r`), либо убрать `\t`/`\r` из набора и признать, что они покрываются транзитивно. Первое честнее спеке.

> ✅ **ИСПРАВЛЕНО 2026-07-14.** `isFormulaRisk` теперь проверяет `raw[0]` без `trimStart`, поэтому лидеры `\t`/`\r` живые. Прогон 16 кейсов зелёный: `\t=1+1`/`\r=1+1`/`\tfoo`/`\rbar` → risk=true; числа (`-5`, `+3.14`, `-1 234,56`, `−0,22`) → risk=false; ведущий пробел `" =1+1"` → risk=false (Excel и так считает это текстом). Комментарий шапки приведён в соответствие.

## (d) 💡 УЛУЧШЕНИЯ / заметки

- **`selection-md.ts:93` (IMG) не проверяет протокол URL, тогда как ветка `A` (`:105`) — проверяет.** `![alt](url)` эмитит `abs(src)` как есть: `javascript:`/`data:` из `img[src]` страницы попадут в сохранённый `.md`. Риск низкий (файл — собственный, `javascript:` в `img` браузеры не исполняют), но несогласованно с гардом ссылок. Добавить тот же `/^(https?:|data:image\/|\/|\.)/i`-фильтр.
- **`export-dialog.ts:84` `rowsToMarkdown` экранирует в ячейках только `|` и `\n`**, тогда как `selection-md.ts` (`escapeMd`) экранирует весь markdown-набор. Значение ячейки `# x` или `**x**` отрендерится как разметка в `.md`-экспорте таблицы. Фиделити, не безопасность.
- **Колонки для сохранения зафиксированы по preview-матрице** (первые `PREVIEW_ROWS+4` строк, `table-extract.ts:474`), а полный файл строится по полной матрице (`export-dialog.ts:604`). Если таблица становится ШИРЕ ниже превью, лишние колонки молча выпадут из файла (`rowsFromMatrix` берёт только `columns`). Редко (таблицы обычно прямоугольны, `buildMatrix` нормализует ширину), но это тихая потеря данных, а не отказ. Стоит хотя бы пересчитывать ширину по полной матрице в `doSave`.
- **`background.ts:221` `downloads.download({saveAs:false})`** для rung 1.5 — молчаливое сохранение в дефолтную папку. Ок по дизайну (§7.3), но тост говорит «Сохранение запущено» без пути — честно (мы и правда не знаем путь без большего доступа).

## (e) ✅ Проверено и в порядке (не переделывать)

- **Оба блокера §G закрыты по-настоящему** (см. вердикт). Rung 3 не кликает якорь → нет тихой навигации; CSV-числа целы, формулы экранированы — прогнано вживую.
- **Манифесты.** Chrome MV3 и Firefox MV2: `permissions` минимальны (`contextMenus, activeTab, scripting, storage, clipboardWrite`), `optional_permissions:["downloads"]`, `host_permissions` нет, `content_scripts` нет, `web_accessible_resources` нет (нет фингерпринт-вектора), `gecko.id` на реальном домене, `gecko_android:{}`, `data_collection_permissions:["none"]`. Ноль install-предупреждений — главный актив цел.
- **Инъекция.** Всё по жесту через `scripting.executeScript` под `activeTab` (`inject.ts`), с MV2-фолбэком на `tabs.executeScript`. Вторая инъекция `xlsx.js` — только по факту выбора xlsx (`background.ts:236`), не динамический `import()` (который упал бы на strict-CSP). `injectFile` — литеральный юнион файлов, опечатка не скомпилируется.
- **XSS/инъекции.** Ноль `innerHTML`/`eval`/`new Function` в исходниках. `el()` кладёт строку только в `textContent` (`overlay.ts:98`). Closed shadow root, `:host{all:initial}`. Превью-таблица — настоящий `<table>` со `scope`, значения ячеек через `textContent` (`export-dialog.ts:442`). `<pre>` сырых байтов — тоже `textContent` (`:463`). Manual-copy textarea — `.value`, не `innerHTML` (`engine.ts:381`).
- **Robustness.** Picker: единый `AbortController`-эквивалент через `trapKeys`/`untrack`/`release` в `finish()` (`picker.ts:183`); capture-фазное глушение `mousedown/click/submit`, чтобы пик не ушёл по ссылке; Esc/Отмена/right-click — три выхода; roving tabindex, 1–9, стрелки; тач через ≥44px кнопки. Двойная инъекция: `overlayExists()` проверяет DOM-маркер, не модульный стейт (`overlay.ts:39`) — свежий модуль-инстанс не плодит второй оверлей. `pagehide` → `leaveSignal()` абортит chunked-экстракцию (`engine.ts:79`), а `file-writer.ts:35` ревокает все blob-URL. Blob-URL ревокаются в 60 с + на `pagehide` (`file-writer.ts:55`) — не сразу после `.click()` (иначе Firefox отменяет загрузку).
- **Патологические таблицы.** `colspan/rowspan` клампятся `MAX_SPAN`/`MAX_COLS` (`table-extract.ts:357`), матрица строится чанками с `yieldToMain()` каждые 200 строк и уважает `AbortSignal` (`:315`), `readCell` использует `textContent`, не `innerText` (нет reflow на ячейку), глубина рекурсии `MAX_CELL_DEPTH=12`. Вложенные таблицы сплющиваются в текст, не «разворачиваются» в строки родителя (`:113`). `>200k` ячеек / `>1048576` строк — xlsx блокируется, предлагается потоковый CSV (`export-dialog.ts:309`).
- **Escape-hatch `save.html`** (`save/App.tsx`) — пересборка Blob на своём origin/CSP; ключ читается один раз и стирается (`:51`), проверяется префикс ключа (`:35`), лимит 8 МБ (`background.ts:248`). Blob-URL ревокается в 60 с.
- **Filename-санитайзер** (`filename.ts`): bidi/RTL-override срезаются (`:74`), C0/C1 контролы, path-сепараторы, `..`-traversal, reserved Windows-имена (`CON`/`PRN`/…), трейлинг-дот/пробел, NFC, кламп 80. Расширение всегда из ФОРМАТА, не из ввода (`file-writer.ts:141`).
- **Честность UI.** Тост говорит «Сохранение запущено» (факт), не «Готово!»; кросс-доменный отказ объясняет причину и предлагает «Открыть»/«Включить разрешение» (`engine.ts:316`); «выделение пропало» вместо пустого файла (`:114`); manual-copy не притворяется, что скопировал (`:372`). Warn-режим guard требует явного чекбокса-подтверждения (`export-dialog.ts:361`).
- **Сеть.** Единственный `fetch` — rung 2 (`file-writer.ts:229`), `credentials:'omit'`, ровно тот ассет, что просил пользователь; подчиняется CORS страницы (host-permissions нет), т.е. проходит только при `ACAO`. Соответствует дизайну §5.9. Аналитики/beacon нет.
