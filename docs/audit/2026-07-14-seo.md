Аудитор: senior code auditor (read-only по `extensions/`, `packages/`). Версия 1.0.0, MV3, Chrome+Firefox, WXT 0.20.27 + React 19 + TS 7.

## (a) Вердикт

Самое чистое из четырёх расширений — заявка PLAN §0 подтверждается кодом. **Блокеров нет.** Разрешения минимальны (`storage` + один `<all_urls>` контент-скрипт; ни `activeTab`, ни `scripting`, ни `host_permissions`, ни `debugger`). **Ключевой XSS-путь (недоверенный контент страницы → привилегированный UI расширения) — чист и доказуемо безопасен:** весь контент страницы рендерится как текстовые дети React (авто-эскейп), единственный не-текстовый сток (`background-image: url()` соц-карточки) присваивается через CSSOM и экранирован. `web_accessible_resources: axe-run.js` — экспозиция безвредна (враждебная страница максимум запустит axe на собственном DOM). `DANGEROUS_EVAL` — это `new Function` внутри бандленного axe-core, раскрыт в STORE.md правдиво и достаточно. Все фиксы §18a/§18c/§18d держатся, регрессий не найдено. Что есть — три пункта класса ⚠️ (устойчивость fetch-ов + доверие к a11y-отчёту как к данным страницы) и мелочи. Плюс дыра в тестах: **DevTools-панель (777 строк) не покрыта ни одним e2e** — как и признано в PLAN §18d.

## (c) ⚠️ ВАЖНОЕ

### 1. Нет тайм-аута и лимита размера на fetch robots.txt / sitemap.xml / HEAD — SEO-скан может зависнуть навсегда
`extensions/seo/utils/indexability.ts:16-24` (`fetchText` → `res.text()`), `:172` (favicon HEAD), `:259` (`xRobotsCheck` HEAD), `:303` (`indexabilityChecks` без обёртки тайм-аутом).
- Единственный тайм-аут в потоке — `AUDIT_TIMEOUT_MS = 30_000` в `content.ts:26`, и он покрывает **только** axe-путь. Путь `extractSeo` → `buildSeoReport` → `indexabilityChecks` (`content.ts:28-32`) не имеет ни `AbortController`, ни лимита на объём тела.
- Same-origin сервер, который отдаёт бесконечный chunked `/robots.txt` или многогигабайтный `/sitemap.xml`, заставит контент-скрипт читать его целиком (`res.text()`) → OOM/зависание, а панель/попап навсегда останутся в состоянии «Reading the page…» без ошибки (спиннер `panel/App.tsx:267`, `popup/App.tsx:127`).
- Почему важно: аудитор по определению запускают на чужих, возможно враждебных/сломанных сайтах. Отказоустойчивость — заявленная цель (комментарий `indexability.ts:4-7` «Every probe is defensive»), но защита неполна: покрыт network-error, не покрыты медленный/огромный ответ.
- Фикс: обернуть каждый fetch в `AbortController` с тайм-аутом (~5-8 c) и читать тело через `Response.body` reader с лимитом (например 512 КБ), обрезая остаток. Общий тайм-аут на `buildSeoReport` тоже не помешает.

### 2. A11y-отчёт — это данные, управляемые страницей; `helpUrl` попадает в `<a href>` без allowlist схемы
`extensions/seo/utils/report-ui.tsx:112-119` (`<a href={violation.helpUrl} target="_blank">`), маппинг без валидации в `utils/a11y.ts:27-35`.
- axe.run исполняется в MAIN-world страницы (`axe-run.ts`), поэтому весь `A11yReport` — по сути недоверенные данные: страница может подменить axe/DOM и повлиять на вывод. Дополнительно её можно **подделать** во время активного аудита (см. §3).
- `help`, `nodes`, `id` рендерятся как текст React (эскейп — безопасно). Но `helpUrl` идёт прямо в `href`. Враждебная страница может подставить произвольный URL → фишинговая ссылка «…— how to fix» в привилегированном UI.
- Исполнение `javascript:` **заблокировано** дефолтным CSP страницы расширения (`script-src 'self'`), так что это НЕ RCE. Остаётся фишинг (http/https) + подрыв доверия к отчёту.
- Фикс: в `a11y.ts` при маппинге отбраковывать `helpUrl`, не начинающийся с `http:`/`https:` (заменять на `''`/дефолт). Трактовать весь `A11yReport` как untrusted.

### 3. Nonce не секретен — комментарий вводит в заблуждение, подделка результата аудита возможна
`extensions/seo/entrypoints/content.ts:54-56` (`makeNonce`, комментарий «unguessable enough to reject casual page-forged messages»), `:90-95` (nonce кладётся в `script.dataset.blurNonce`, `keepInDom: true`).
- Nonce записывается в атрибут DOM-элемента `<script>` в MAIN-world и остаётся там до cleanup. Страница читает его синхронно (например MutationObserver на вставку скрипта), после чего может `window.postMessage({source:'blur-seo-a11y', nonce:<украден>, ok:true, report:<подделка>})` — контент-скрипт примет (`content.ts:74-81`: `event.source===window`, `isA11yResult`, совпадение nonce), т.к. слушатель активен во время аудита. axe.run асинхронен → страница успевает выиграть гонку.
- Итог: враждебная страница во время аудита может (а) скрыть реальные нарушения / показать фейковые (пользователь поверит «0 нарушений»), (б) подсунуть `helpUrl` (см. §2).
- Не блокер: требует, чтобы пользователь сам запустил аудит на враждебной странице; `javascript:` заблокирован CSP. Но комментарий про «unguessable» ложен — nonce страница видит.
- Фикс: минимум — §2 (санитизация helpUrl) + честный комментарий; идеально — `use_dynamic_url: true` (см. 💡) и валидация формы отчёта.

## (d) 💡 УЛУЧШЕНИЯ

- **`use_dynamic_url: true` для WAR.** `wxt.config.ts:68-70` объявляет `axe-run.js` со стабильным URL `chrome-extension://<id>/axe-run.js`. Любой сайт может (а) зафингерпринтить установку расширения, пробуя загрузить ресурс, (б) сам предзагрузить runner. `use_dynamic_url: true` ротирует URL за сессию → защита от фингерпринтинга и предзагрузки. WXT `injectScript` умеет резолвить динамический URL.
- **CSP-ограничение axe (UX).** На страницах со строгим CSP (`script-src 'self'` без extension-origin / без `unsafe-eval`) инжект `axe-run.js` блокируется или `Function(` в axe бросает → аудит падает. Падение обрабатывается (ошибка всплывает в UI), но сообщение общее — пользователь на GitHub/строгих сайтах увидит «This page cannot be audited…». Стоит распознавать CSP-отказ и давать точную формулировку.
- **Синхронный reflow на каждом extract.** `extract-seo.ts:48-62` (`getBoundingClientRect` на каждом `a/button/input/...`) и `:65-69` (`document.body.innerText`) форсируют layout/reflow. На огромных DOM — джанк. Приемлемо, но можно гейтить (напр. пропускать tap-target-подсчёт при >N элементов).
- **Глубоко вложенный JSON-LD → переполнение стека.** `checks.ts:217` (`visit` рекурсивна без лимита глубины). Патологический вложенный `@type` даст `RangeError`; он ловится выше и всплывает как ошибка, так что косметика. Депт-кап (напр. 32) убрал бы риск.
- **TS `^7.0.2`** (`package.json:27`) — bleeding edge (нативный компилятор). Для воспроизводимой AMO sources-сборки лучше пинить точную версию.

## (e) ✅ Проверено и в порядке

- **XSS-путь (контент страницы → привилегированный UI) ЧИСТ.** Grep по всему `extensions/seo`: **ноль** `innerHTML`/`outerHTML`/`insertAdjacentHTML`/`dangerouslySetInnerHTML`/`document.write`/`eval`/`new Function`. Весь контент страницы (title, meta, headings, JSON-LD `@type`, hreflang, link href, og:* поля) рендерится текстовыми детьми React (авто-эскейп): `panel/App.tsx` (meta `dd`, outline, hreflang-таблица, SERP, SocialCard title/desc/host), `popup/App.tsx`, `report-ui.tsx`. Единственный не-текстовый сток — `SocialCard` `style={{ backgroundImage: `url("${previewImage.replace(/"/g,'%22')}")` }}` (`panel/App.tsx:523`): присваивается через React style-объект = CSSOM single-property setter (нельзя внедрить лишние декларации), кавычки → `%22`, `url(javascript:)` не исполняется, URL уже прошёл `new URL().href` (`extract-seo.ts:93-100`). Не эксплуатируется.
- **WAR-экспозиция безвредна.** Враждебная страница, загрузившая `axe-run.js` сама: `document.currentScript.dataset.blurNonce` пуст → axe запускается на **её собственном** DOM и `postMessage` в её же window с `nonce:''` (`axe-run.ts:14-32`). Никаких привилегий расширения, никакой утечки. Контент-скрипт игнорирует чужой nonce (`content.ts:77`). Всё, что могла бы сделать страница, — забандлить axe сама.
- **`DANGEROUS_EVAL` — правда и раскрыт достаточно.** Собранный `.output/chrome-mv3/axe-run.js` (584 КБ) содержит 14 `Function(` (Vite переминифицировал `new Function`→`Function(`), 0 `eval(`; источник — axe-core 4.12.1 (doT-шаблонизатор + компиляция сообщений/функций). В собственном коде seo — ноль eval-стоков. Runner исполняется в контексте **страницы** под её CSP, не под CSP расширения. Reviewer note STORE.md §2 фактически верна и достаточна.
- **§18a-фиксы держатся:** background отвечает через `sendResponse + return true`, не возвращает Promise (`background.ts:100-117`), то же в контент-скрипте (`content.ts:112-141`); robots `content="none"` = noindex (`extract-seo.ts:133-138`); относительный og:image резолвится (`extract-seo.ts:93-100, 207-217`); верхнеуровневый JSON-LD-массив обходится (`checks.ts:223-226`); `Disallow: /` матчится только в группе `User-agent: *` (`indexability.ts:33-56`); X-Robots HEAD best-effort, не-200 → `null` (`indexability.ts:263`); canonical http↔https через нормализацию протокола (`checks.ts:296`).
- **§18c-фикс держится (ключевая регрессия):** `max-image-preview:none` НЕ читается как noindex — токенизация по имени директивы до `:` в мета (`extract-seo.ts:133-138`) и в X-Robots (`indexability.ts:236-241`). `url()` соц-превью с кавычками экранируется (`panel/App.tsx:523`). Sitemap с чужого origin → нейтральная заметка без fetch (`indexability.ts:110-124`). Множественный canonical детектится (`extract-seo.ts:109-115`, `checks.ts:380-388`).
- **§18d-фикс держится:** вложенные JSON-LD-сущности валидируются, но в список попадают только при проблеме; верхнеуровневые — всегда (`checks.ts:217, 251`). Подтверждено unit-тестом (`e2e/seo/unit.spec.ts:41-60`).
- **Разрешения минимальны, честно.** Собранный манифест: `permissions:["storage"]`, WAR только `axe-run.js`, `devtools_page`, один content_script `<all_urls>` document_idle. Ни `activeTab`/`scripting`/`host_permissions`/`debugger`. gecko id `seo@blockaly.com` стабилен, `data_collection_permissions.required:["none"]` — честно: fetch robots/sitemap/favicon/HEAD — same-origin, `credentials:'omit'`, ничего не уходит разработчику/третьим лицам, это не «сбор данных» по CWS/AMO. Политика CWS 2026-08-01 (strictly-necessary + in-UI disclosure) для seo не триггерится: сетевых передач наружу нет. Самое чистое из четырёх — подтверждено.
