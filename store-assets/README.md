# Store assets

Visual assets for the Chrome Web Store and Firefox AMO listings, for the four
Blockaly extensions.

Everything in this directory is **generated**, not hand-drawn:

```bash
npm run icons         # extensions/<name>/public/icon/{16,32,48,128}.png
npm run store-assets  # store-assets/<name>/promo-tile-440x280.png
```

Both scripts use `scripts/lib/draw.mjs` — a dependency-free software rasterizer
and PNG encoder (no sharp / canvas / jimp; nothing to install). Edit the artwork
there and re-run the scripts; never hand-edit the PNGs, they will be overwritten.

## What is MISSING — a human must produce these

### 1. Screenshots — REQUIRED by Chrome, BLOCKING for every listing

**These cannot be generated by any script in this repo.** A screenshot has to
show the real extension running: the popup, the options page, or (for `perf` /
`seo`) the DevTools panel, against a real web page. That means loading the built
extension into a real browser and capturing the UI. No amount of procedural
drawing substitutes for it, and faking one would be a store-policy problem.

Chrome requires **at least 1** and allows **up to 5** screenshots, at exactly
**1280x800** or **640x400** px (24-bit PNG, no alpha). AMO wants at least one as
well (no fixed size, 1280x800 works).

To capture them:

1. `npm run build:<name>` (or `npm run build`) — output lands in
   `extensions/<name>/.output/chrome-mv3/`.
2. Load that folder via `chrome://extensions` -> Developer mode -> *Load unpacked*.
3. Open the surface you want to show, resize the window so the capture is
   1280x800, and screenshot it.
4. Save as `store-assets/<name>/screenshot-1-<what>.png` etc.

Suggested shot list, per extension:

| Extension | Screenshots to capture |
|---|---|
| blur | Popup with blur toggles on a page with images/video blurred; options page with the text-match rules |
| adblock | Popup showing the blocked-count for the current site; options page with filter lists |
| perf | DevTools panel with Core Web Vitals + the network/page-weight table on a real site |
| seo | DevTools panel showing the meta/heading audit and the axe-core accessibility findings |

### 2. Marquee promo tile — 1400x560 (OPTIONAL, Chrome only)

Not generated. Only needed if the listing is ever considered for featuring.
Same brand language as the 440x280 tile; produce in a real design tool.

### 3. A designer pass on the generated art — RECOMMENDED before submit

The generated assets are *deliberate*, not placeholders, but they were produced
by code, and **nobody has looked at them yet**:

- **The icons have not been reviewed by a human at 16px in a real toolbar.**
  Somebody must load all four extensions and confirm each mark is legible and
  distinguishable at 16px on both a light and a dark toolbar before submitting.
- **The type on the promo tiles is drawn with a hand-built geometric stroke
  font** (`GLYPHS` in `scripts/lib/draw.mjs`) — there is no font rasterizer
  available in this repo. It is legible, but it is not a real typeface: spacing
  is mechanical and the `&` glyph is the weakest character. If the listing is
  meant to look professionally designed, **rebuild the four tiles in a design
  tool with the real brand typeface** and drop them in over the generated ones.
  The generated tiles are good enough to unblock submission, not to win a design
  award.

## Design system (so replacements stay on-brand)

Shared across all four marks:

- Full-bleed rounded square, corner radius `0.22 x size`, 2% inset.
- Background: diagonal gradient from a light to a deep tint of the product hue,
  plus a soft top-left sheen. Saturated mid-tones only — never near-white or
  near-black, so the icon holds up on both light and dark toolbars.
- Mark: white, bold, few shapes, no thin strokes, no text. Designed at 16px
  first, then scaled up.

| Extension | Hue | Base | Silhouette |
|---|---|---|---|
| blur | Blue | `#2563eb` | Crisp disc dissolving into a soft gradient halo |
| adblock | Red | `#dc2626` | Shield with a knocked-out diagonal slash |
| perf | Green | `#10a35a` | Speed gauge: fat arc + chunky needle |
| seo | Purple | `#7c3aed` | Magnifier: heavy ring + diagonal handle |

Promo tiles put that mark on a dark slate field with a brand-colored glow, the
product name in white, and the `Blockaly` publisher line beneath it.
